# Upstream v2.6 Poetrieve Localization Design

## Goal

Bring `buri34/poenavi` v2.6.0 into the localization fork while preserving
the fork's English interface, English PoE and PoE 2 guide data, authoritative
terminology checks, localization documentation, and locale validation.

## Integration Strategy

Merge `upstream/main` into the localization fork's `main` branch with a merge
commit. Do not rebase or cherry-pick the 190-commit Poetrieve development
history. A normal merge preserves upstream ancestry, incorporates all v2.6.0
fixes, and avoids rewriting the branch used by the open pull request.

The merge must retain localization-only files even though they do not exist on
upstream:

- `README.en.md`
- `guide_data_en.json`
- `guide_data_poe2_en.json`
- `data/authoritative_guide_terms.json`
- `data/i18n/en.json`
- `data/i18n/ja.json`
- `data/i18n/ui_en.json`
- `data/i18n/ui_ja.json`
- `docs/localization.md`
- `scripts/validate_locales.py`
- localization and authoritative-terminology tests

## Translation Boundary

The English product name for upstream's `ぽえとれ` feature is **Poetrieve**.
Use `Poetrieve` for the window title, settings section, and documentation.
Use direct functional English for controls and messages, including `Price
Check`, `Search`, `Official Trade`, and `Price-check hotkey`.

Translate all user-visible Poetrieve strings through the existing `tr_ui`
catalog mechanism. This includes labels, buttons, tooltips, notices, errors,
filter descriptions, search status, league and currency choices, and dynamic
messages.

Do not translate or remove Japanese strings that function as parser tokens,
metadata keys, official Trade API lookup data, item-copy markers, or test
fixtures. Those strings are operational data required to recognize Japanese
PoE item text. Preserve upstream's English parser aliases so both Japanese and
English item-copy formats continue to work.

Use official Path of Exile terminology for game concepts. Examples include
`Chaos Orb`, `Divine Orb`, `Foulborn`, `Blighted Map`, `Cluster Jewel`,
`Fractured`, `Shaper`, `Elder`, `Crusader`, `Hunter`, `Redeemer`, and
`Warlord`. If a newly introduced game term cannot be resolved from upstream's
English aliases, existing authoritative term data, official trade metadata, or
established PoE usage, ask the user with enough UI or gameplay context to
choose an English rendering.

## Feature Integration

Integrate upstream's Poetrieve implementation without removing functionality:

- Windows item-copy capture and configurable hotkey
- Japanese and English item parsing
- official Path of Exile Trade API searches
- poe.ninja reference prices and trend display
- category-specific search presets and filters
- league, listing-status, currency, corruption, item-level, gem, socket,
  influence, and modifier controls
- cursor-side window positioning and focus behavior
- metadata build and validation scripts
- release packaging for Poetrieve metadata and influence icons

The feature must remain lazy-loaded as upstream designed it so startup behavior
and non-Windows test environments are not regressed.

## Configuration and Compatibility

Merge v2.6.0's new Poetrieve configuration into the localized defaults and
configuration manager. If the upstream schema version collides with a schema
version already used by the localization fork, advance the fork schema and add
a migration regression. Existing user settings and locale selection must
survive the upgrade.

Retain locale-aware guide loading, fallback behavior, caching, editor save
targets, segment-recorder persistence, and translated detached-panel titles
from the previous upstream integration.

## Documentation

Preserve the Japanese README changes and add equivalent Poetrieve information
to `README.en.md`. Document the default hotkey, item-copy workflow, supported
price sources, official Trade link, network access, current limitations, and
the Poetrieve name consistently.

Do not translate internal developer audits or Japanese parser sample data
unless they contain strings displayed by the released application.

## Verification

Before committing the completed merge:

1. Add failing localization regressions for new user-visible strings and any
   configuration migration collision.
2. Resolve merge conflicts while preserving both upstream behavior and the
   localization architecture.
3. Run focused Poetrieve parser, trade, UI, hotkey, metadata, and configuration
   tests.
4. Run `python scripts/validate_locales.py`.
5. Run the complete pytest suite in an isolated temporary user-data directory
   with Qt in offscreen mode.
6. Run `git diff --check`.
7. Request an independent code review and address Critical or Important
   findings with regression tests.

The final merge commit must have `upstream/main` as an ancestor, preserve all
localization resources, and leave unrelated untracked files untouched.
