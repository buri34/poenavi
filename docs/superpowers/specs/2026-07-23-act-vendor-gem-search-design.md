# Act別ショップジェム検索 設計

## 目的

PoBから解決済みのジェム取得計画を使い、現在Actでショップから購入できるジェムだけを、ショップ検索に適した短縮語のOR検索として入力する。

## 対象と選択フロー

- 対象は、PoBインポート済みのPoE1ジェム取得計画である。
- 現在Actは既存のエリア移動連動をそのまま使う。
- 長押し時に現在Actの全エントリを集計するため、ジェム行やクエスト見出しを事前に選択する操作は追加しない。
- `type` が `vendor` または `lilly` のジェムだけを対象にする。`quest` は常に除外する。
- 購入対象がない場合は、クリップボード・前面ウィンドウ・キー送信を変更しない。

## 検索文字列

- ジェムの識別子はPoBの `nameSpec` と `gems.json` の英語キーを照合して得た解決済みの `name` を使う。
- ショップ検索に使う日本語短縮語は専用の明示的な対応データで管理する。推測変換はしない。
- 例: `momentum support` → `モーメン`、`precision` → `プレシジ`、`leap slam` → `リープス`、`spectral throw` → `ラルスロ`、`chance to bleed support` → `出血付与`、`frostblink` → `ストブリ`。
- 同じAct内の対象短縮語は、取得計画の順序を保ちつつ重複を除き、`|` で連結する。
- 対応データにない購入ジェムは検索対象から除外して診断ログに残す。正式名へのフォールバックはしない。

## キー入力とWindows安全性

- 初期キーは `CapsLock`。既存設定画面の `HotkeyButton` で変更可能にする。
- `pynput` リスナーで押下・解放を監視し、0.4秒の長押しで一度だけ実行する。短押しではPoENaviの操作を行わない。
- 同じ押下中のリピート、貼り付け処理中の再実行、キー解放後のタイマー発火を防ぐ。
- Windowsで前面ウィンドウがPath of Exileプロセスであることを確認できる場合だけ実行する。検証不可・非Windows・ゲーム非アクティブ時は入力しない。
- 既存の `paste_text_to_poe_search` を再利用する。これはQtクリップボードへ検索文字列を保存後、既存のWin32前面化処理でゲームへ戻り、`pynput` の `Ctrl+F` と `Ctrl+V` で検索欄へ貼り付ける。IMEに依存する直接入力は使わない。

## テスト

- 購入対象の抽出、報酬除外、短縮語の順序・重複排除・未定義除外をユニットテストする。
- 長押し判定の短押し無操作、一回だけの実行、無対象・非PoE前面時の無操作をテストする。
- 設定の初期値・保存と、既存設定との後方互換をテストする。
- Qtテストは `QT_QPA_PLATFORM=minimal` を使い、全体テストは `-p no:unraisableexception` を付ける。
